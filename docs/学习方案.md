# 📡 流媒体 / 推流系统学习方案（面向无人机工程师）

> 目标：从「靠 AI 改代码」升级到「能独立设计、调试、优化一条推流链路」，重点是无人机图传场景。

---

## 一、知识地图（全景）

任何一条推流链路都由 5 层构成，无人机场景每层都有特殊考点：

| 层 | 内容 | 无人机特殊考点 |
|---|---|---|
| ① 采集层 | 摄像头 → ISP → YUV/RGB 原始帧 | 载荷功耗、传感器噪声 |
| ② 编码层 | H.264/H.265、I/P/B 帧、GOP、NALU、码率控制 | H.265 压缩比高但硬解兼容性 |
| ③ 封装/协议层 | MP4/FLV/MPEG-TS、RTP/RTSP/RTMP/HLS/SRT/WebRTC | 低延迟首选 RTSP/SRT/WebRTC |
| ④ 传输/弱网层 | TCP vs UDP、ABR 码率自适应、FEC、丢包重传、抖动缓冲 | 4G/5G 弱网、高丢包、带宽波动 |
| ⑤ 解码/渲染层 | 硬解/软解、音视频同步、渲染 | 地面站/APP 端零延迟渲染 |

**断点定位**：大概率在 ②（码流结构）和 ④（弱网对抗），这是重点补的。

## 二、三个候选方案

### 🅰️ 方案 A：理论筑基派（自底向上）

```
阶段1 基础概念 (2周)  → 阶段2 编解码原理 (3周)
阶段3 协议与封装 (3周) → 阶段4 框架实战 (4周)
阶段5 无人机图传综合 (4周)
```
- 产出：能画完整码流结构图，讲清 RTSP 握手全过程
- 周期：约 4 个月
- 适合：长期扎根音视频，不怕前期枯燥
- 风险：见效慢，易放弃

### 🅱️ 方案 B：工程实战派（逆向驱动）⭐ 推荐起点

```
阶段1 抓取手头推流代码逐行搞懂 (1-2周)
阶段2 用 FFmpeg 命令行复刻同样链路 (2周)
阶段3 用 FFmpeg API / GStreamer 写最小 Demo (3周)
阶段4 弱网/延迟专项优化实验 (3周)
```
- 产出：能独立改、调、排障现有推流代码
- 周期：2 个月见效
- 适合：代码在手但不懂底层的现状
- 风险：知识碎片化，需事后补理论（回到方案 A）

### 🅲️ 方案 C：无人机图传专项派（场景聚焦）

```
阶段1 图传链路全景 + DJI/开源方案调研 (1周)
阶段2 低延迟编码 (H.265 + 零延迟 preset) (2周)
阶段3 SRT/WebRTC 弱网传输实战 (3周)
阶段4 端到端 latency 打点与优化 (3周)
```
- 产出：一条可量化延迟的无人机图传 Demo
- 周期：约 2.5 个月
- 适合：想直接出工作成果
- 风险：跳过通用协议，换场景要重学

---

## 三、推荐路径

**方案 B 起步 → 方案 A 补全 → 方案 C 出成果**

先用手头真实代码建立「为什么」，比啃纯理论效率高 3 倍。
