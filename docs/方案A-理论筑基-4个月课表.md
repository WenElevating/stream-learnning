# 🎯 方案 A：理论筑基派 · 4 个月分周课表

> 自底向上系统学，目标：4 个月后能独立画出一条完整推流链路，讲清每一层原理，并能用 C++ + FFmpeg 写出最小推流 Demo。
> 技术栈：C++（推流核心）+ FFmpeg。C# UI 业务暂不涉及。
> 配套：每个阶段都有「手头真实推流代码对照」环节，理论立刻落地到你的工程。

---

## 阶段 1｜基础概念（第 1–2 周）

**目标**：建立音视频整体心智模型，不再"知其然不知其所以然"。

| 周 | 主题 | 关键产出 |
|---|---|---|
| W1 | 像素、分辨率、帧率、码率、YUV/RGB、色彩空间 | 能解释为什么摄像头出 YUV 不出 RGB |
| W1 | 封装 vs 编码 的区别（容器/格式/编码 三者关系） | 能分清 .mp4 / .flv / .ts 是封装，H.264/H.265 是编码 |
| W2 | 音视频同步、PTS/DTS、时间戳 | 能讲清为什么音视频不同步、DTS≤PTS 的原因 |
| W2 | 工具：MediaInfo / ffprobe 看懂一段码流参数 | 用 ffprobe 解析你手头推流代码产出的码流 |

**对照环节**：用 `ffprobe` 打开你手头的推流输出，逐字段解释每个参数含义。

---

## 阶段 2｜编解码原理（第 3–5 周）⭐ 核心

**目标**：看懂 H.264 码流的二进制结构，这是你最薄弱也最重要的断点。

| 周 | 主题 | 资源 | 关键产出 |
|---|---|---|---|
| W3 | 帧内预测 / 帧间预测、运动估计、残差 | 雷霄骅《H.264视频编解码分析》目录 | 能解释压缩是怎么发生的 |
| W3 | I/P/B 帧、IDR、GOP、GOP 的 M/N 参数 | 知乎 H.264 编码基础 | 能讲清为什么切到某个 GOP 才能随机定位 |
| W4 | NALU 结构、起始码、SPS/PPS | 极客时间《05 码流结构》 | 能从一段 .h264 二进制里找到 SPS |
| W4 | 码率控制 CBR/VBR/CRF、zero-latency preset | FFmpeg 文档 | 知道无人机为什么用 zero-latency |
| W5 | H.265 vs H.264、AV1/HEVC 选型 | 自查 | 能讲清无人机为何上 H.265 以及硬解坑 |

**动手**：用 FFmpeg 把一段视频导出为纯 H.264 裸流（`.h264`），用十六进制查看器找 NALU 起始码和 SPS。

## 阶段 3｜封装与协议（第 6–8 周）

**目标**：搞懂 RTSP / RTMP / SRT / WebRTC 各自的建联和数据传输过程。

| 周 | 主题 | 资源 | 关键产出 |
|---|---|---|---|
| W6 | 封装格式：MP4 / FLV / MPEG-TS 的 box/包结构 | FFmpeg wiki | 知道为什么直播用 FLV/TS 不用 MP4 |
| W6 | RTP / RTCP：报文结构、时间戳、SSRC | 雷霄骅 RTP 系列 | 能画 RTP 头部 12 字节 |
| W7 | RTSP 完整握手：OPTIONS→DESCRIBE→SETUP→PLAY | 阿里云《RTSP协议抓包及讲解》 | 能复述 RTSP 4 步握手每步做什么 |
| W7 | Wireshark 抓包 RTSP 实操 | 断点实验室 rtsp 协议格式解析 | 抓你手头的推流，对照报文 |
| W8 | RTMP / HLS / SRT / WebRTC 选型对比 | 知乎《流媒体协议全面分析》 | 能给无人机场景选型并说明理由 |
| W8 | SRT 原理（ARQ、FEC）、WebRTC ICE/SDP | SRT Alliance + WebRTC for the Curious | 讲清 SRT 为什么比 RTMP 低延迟 |

**对照环节**：Wireshark 抓你手头推流的建联过程，逐行对照课表里的握手说明。

---

## 阶段 4｜FFmpeg 框架实战（第 9–12 周）

**目标**：从"用命令行"升级到"用 C++ API 写推流器"。

| 周 | 主题 | 资源 | 关键产出 |
|---|---|---|---|
| W9 | FFmpeg 命令行：采集/转码/推流/拉流 | FFmpeg 官方 wiki | 用一条命令推 RTMP 流到 SRS |
| W9 | 编译 FFmpeg（Windows/MSVC 或 vcpkg） | 自查 | 在你的机器跑通 `ffmpeg -version` |
| W10 | 核心数据结构：AVFormatContext / AVCodecContext / AVPacket / AVFrame | 0voice FFmpeg API 文档 | 画出 4 个结构的关系图 |
| W10 | 解码流程：avformat_open_input → 解码一帧 | 博客园《FFmpeg使用指北-视频解码》 | C++ 写最小解码器，输出一帧 YUV |
| W11 | 编码流程：YUV → H.264 → AVPacket | 雷霄骅 simplest_ffmpeg_encoder | C++ 写最小编码器 |
| W12 | 推流：avformat_alloc + av_interleaved_write_frame 到 RTMP/RTSP | 雷霄骅 simplest_ffmpeg_streamer.cpp | **复刻你手头的推流代码** |

**里程碑**：第 12 周用纯 FFmpeg API 复刻你手头推流代码的核心链路，这是阶段验收点。

## 阶段 5｜无人机图传综合（第 13–16 周）

**目标**：把前 4 个阶段的知识应用到无人机场景，做端到端优化。

| 周 | 主题 | 关键产出 |
|---|---|---|
| W13 | 无人机图传链路全景 + DJI / 开源方案调研 | 画出你的项目实际链路图 |
| W13 | 低延迟编码调优（tune zerolatency、GOP=1、无 B 帧） | 量化编码延迟 |
| W14 | 弱网对抗：FEC、丢包重传、抖动缓冲、ABR | 讲清每种手段的代价 |
| W14 | SRT/WebRTC 实战（payload 端推流 + 地面端拉流） | 跑通端到端 |
| W15 | 端到端 latency 打点（采集→编码→网络→解码→渲染） | 拿到每段延迟数字 |
| W16 | 针对最大延迟段做优化、总结复盘 | 输出一份优化报告 |

**最终产出**：一条可量化延迟的无人机图传最小链路 + 复盘报告。
